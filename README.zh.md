# spec-kit

一個 Claude Code 技能，引導規格驅動開發 (SDD) -- 將功能描述轉化為結構化規格文件、實作計畫和可執行的任務清單。

## 說明

此技能將 [GitHub Spec Kit](https://github.com/github/spec-kit) 的方法論改編為 Claude Code 技能。提供完整的 SDD 工作流程：撰寫功能規格、釐清模糊之處、產生實作計畫、將計畫拆解為依賴排序的任務、分析跨產出物的一致性，以及逐階段執行實作。

## 功能特色

- 將自然語言功能描述轉化為結構化規格文件
- 透過針對性問題引導反覆釐清模糊之處
- 產生包含研究、資料模型和 API 合約的技術實作計畫
- 將計畫拆解為按使用者故事組織的依賴排序、可並行化任務清單
- 執行跨規格、計畫和任務的非破壞性一致性分析
- 按任務順序執行實作並追蹤進度
- 建立和管理專案憲法以規範架構原則
- 產生需求品質檢查清單（「英文的單元測試」）

## 安裝

將技能目錄複製到 Claude Code 技能資料夾：

```
cp -r spec-kit ~/.claude/skills/
```

放置在 `~/.claude/skills/` 的技能會被 Claude Code 自動發現，無需額外註冊。

## 使用方式

透過要求 Claude Code 處理規格文件來觸發此技能。範例提示：

- 「幫我撰寫一個即時聊天系統的規格文件」
- 「建立使用者驗證功能的產品規格」
- 「為聊天功能產生使用 Node.js 和 WebSocket 的技術規格實作計畫」
- 「將計畫拆解為任務」
- 「分析規格、計畫和任務的一致性」

技能遵循 SDD 工作流程：規格 -> 釐清 -> 計畫 -> 任務 -> 分析 -> 實作。
